Universal Exchange Adapters (UEH)

UEH is survivability-first infrastructure for operating against unstable exchange APIs. Not signals. Not financial advice. No-exec by default.

Story in 7 lines

  1. We started by building bots and chasing “wins”.
  2. Exchange APIs didn’t behave like stable contracts—latency, limits, and silent drift were normal.
  3. We learned that correct code can still fail when responsibility boundaries are unclear.
  4. The turning point was naming the boundary: decision → risk_scan, execution gate → exec_gate, orders → worker, exchange differences → adapters.
  5. UEH stopped being a bot project and became survivability-first infrastructure.
  6. Public artifacts are proofs (NO-EXEC); private delivery is controlled (constraints + ops), with clear refusal rules.
  7. The goal is stable operations, clear accountability, and safe delivery boundaries—no profit claims.
